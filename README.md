## Gestion des commits et versions

Nous utilisons la convention **Conventional Commits** :

- `feat:` pour les nouvelles fonctionnalités
- `fix:` pour les corrections de bugs
- `docs:` pour la documentation
- `chore:` pour les tâches techniques

Le versionnage suit **Semantic Versioning (SemVer)** :

- MAJOR.MINOR.PATCH
- Exemple : 1.0.0, 1.1.0, 1.1.1
- Chaque version stable est taguée dans Git
